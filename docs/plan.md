# 🛠️ 急救先锋 — 开发计划方案

## 技术选型
| 层级 | 技术 | 理由 |
|------|------|------|
| 结构 | HTML5 | 语义化标签，单页应用结构 |
| 样式 | CSS3（Flex/Grid） | 响应式布局，动画过渡，无需框架 |
| 逻辑 | Vanilla ES6+ | 轻量、零依赖、快速加载、黑客松高效开发 |
| 数据 | LocalStorage | 本地持久化，无后端依赖，即开即玩 |
| 动画 | CSS Animation + JS | 流畅交互，段位升级特效 |

---

## 项目结构
```
PublicWelfareGame/
├── index.html          # 单页入口，所有页面结构（section切换）
├── css/
│   └── style.css       # 全局样式、响应式、动画
├── js/
│   ├── data.js         # 题目数据、关卡配置、段位规则
│   ├── storage.js      # LocalStorage 封装（读写、排行榜）
│   ├── game.js         # 游戏核心逻辑（计时、计分、答题流程）
│   └── app.js          # UI 渲染、事件绑定、页面路由切换
└── docs/
    ├── requirements.md # 需求文档
    └── plan.md         # 本计划文档
```

## 核心算法草稿

### 得分算法
```
基础分 = 答对题数 × 20
时间奖励 = 剩余秒数 × 2
连击奖励 = 最大连续答对数 × 5
关卡总分 = 基础分 + 时间奖励 + 连击奖励
小红花累计 += 关卡总分
```

### 星级评价（每关）
- 3星：全部答对且剩余时间 > 30秒
- 2星：全部答对或正确率 >= 80%
- 1星：正确率 >= 60%
- 未通过：正确率 < 60% 或时间耗尽
